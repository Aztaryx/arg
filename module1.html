Module 1: HTML Basics

1. HTML Headings

We have 1–6 headings ranging from the most important text (largest heading size) to the least important text (smallest heading size).

  • <h1> defines the most important heading.
  • <h6> defines the least important heading.

Example Code:

bash — 80×24 
<h1>This is heading 1</h1>
<h2>This is heading 2</h2>
<h3>This is heading 3</h3>

2. HTML Paragraphs

Paragraphs are defined with the <p> tag.

Example Code:

bash — 80×24 
<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

Combined Example (Full Structure):

bash — 80×24 
<!DOCTYPE html>
<html>

<head>
<title> My Website </title>
</head>

<body>

<h1>Welcome to my homepage!</h1>
<p>My first website</p>

</body>
</html>

3. HTML Images

HTML images are defined with the <img> tag. Unlike other HTML tags, it has no closing tag like </img>.

Attributes:

  • src: Specifies the path to the image file.
  • alt: Alternative text used to describe the image if it doesn't load.
  • width/height: Sets the size (number or percentage) of the image.

Example Code:

bash — 80×24 
<img src="flower.jpg" alt="Pink tulips" width="104" height="142">

Note: The image file must be .jpg for this specific example code to work successfully.

4. HTML Videos

The <video> tag embeds video content in a document. It holds one or more <source> tags with different video sources; the browser uses the first it supports.

Supported Formats: MP4, WebM, and OGG.

Example Code:

bash — 80×24 
<video width="320" height="240" controls>
  <source src="assets/EDGE_OF_INPHINITY.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>

The text between the video tags displays only in browsers that don't support the video element.

Attributes:

  • src: specifies the URL of the video file
  • width: sets the width of the video
  • muted: mutes the audio
  • loop: restarts the video once it finishes
  • autoplay: plays the video automatically on page load

5. HTML Audios

The <audio> tag embeds sound content in a document. Like the video tag, it holds one or more <source> tags, and the browser chooses the first source it supports.

Supported Formats: MP3, WAV, and OGG.

Example Code:

bash — 80×24 
<audio controls>
  <source src="horse.ogg" type="audio/ogg">
  <source src="horse.mp3" type="audio/mpeg">
  Your browser does not support the audio tag.
</audio>

Attributes:

  • src: specifies the URL of the audio file
  • muted: mutes the audio
  • loop: restarts the audio once it finishes
  • autoplay: plays automatically on page load

6. HTML Links

HTML links are hyperlinks defined with the <a> tag (anchor tag) and the href attribute (Hypertext Reference) for linking other webpages to a document.

Example Code:

bash — 80×24 
<a href="https://www.google.com">This is a link</a>
  • The link's destination is specified in the href attribute.
  • Attributes are generally used to provide additional information about HTML elements.

7. HTML Forms — Part 1

The most used form element is <input>. Its type attribute specifies what kind of input to display.

Advance Lesson: External CSS (Preview for Module 2)

This is most recommended when you are fully using CSS to design your website. CSS is written in a separate file that you only need to link to your HTML file.

External styles are defined within the <link> element, inside the <head> section of an HTML page.

HTML Example:

bash — 80×24 
<!DOCTYPE html>
<html>
<head>
<link rel="stylesheet" href="mystyle.css">
</head>
<body>

<h1>This is a heading</h1>
<p>This is a paragraph.</p>

</body>
</html>

CSS File Example (mystyle.css):

bash — 80×24 
body {
  background-color: lightblue;
}

h1 {
  color: navy;
  margin-left: 20px;
}

Key Notes:

  • The href attribute filename uses the .css extension, marking it as a CSS-only file.
  • Both internal and external CSS go inside the <head> tag of the HTML file.
  • rel="stylesheet" is required to connect the HTML file to the external CSS file.
